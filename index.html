// Implementation

Three-Phase Build

PHASE 01
Federated Learning Baseline
Built a CNN model trained across 5 clients on MNIST using the FedAvg algorithm. No raw data sharing — only model weights exchanged each round.
96.20% accuracy · 10 rounds · 5 clients
PHASE 02
Blockchain Integration
Deployed a Solidity smart contract on a local Ethereum network (Ganache). Every model update is hashed and logged as an immutable on-chain transaction via Web3.py.
50 on-chain updates · Smart contract verified
PHASE 03
Privacy + Attack Detection
Added Differential Privacy (Gaussian noise), cosine similarity trust scoring, and Proof-of-Contribution consensus. Simulated a poisoning attack — caught and blocked by round 2.
Attacker rejected round 2 · 96.78% final accuracy

// System Design

Architecture Overview

🖥
5 FL Clients
Each holds 12,000 private MNIST images. Trains locally, never shares raw data.
⚙
Federated Server
Runs FedAvg aggregation. Only processes blockchain-verified updates.
⛓
Ethereum Blockchain
Local Ganache network. Smart contract logs every update immutably.
🛡
Trust Scoring (PoC)
Cosine similarity tracks client honesty. Malicious clients decay below threshold and are blocked.

FedAvg Formula

# Global model update each round w_global = sum(p_i * w_i) # p_i = data proportion of client i p_i = m_i / sum(m_i) # Local SGD update w_i(t+1) = w_i(t) - η∇L_i(w_i(t))

Trust Score Update

# Proof-of-Contribution scoring T_i(t+1) = α·T_i(t) + (1-α)·S_i(t) # S_i = cosine similarity S_i = dot(w_global, w_local) / (‖w_global‖ · ‖w_local‖) # Reject if below threshold if T_i < 0.85: reject(client_i)
// Attack Simulation

Round-by-Round Timeline

ROUND 1
Attacker submits poisoned weights
Trust score: 0.900 — still above threshold. Accepted. Model corrupted, accuracy drops to 4.3%.
ROUND 2
Attacker REJECTED — global model reset
Trust score: 0.810 — below 0.85 threshold. Blockchain logs rejection. Model reinitialised from clean state.
ROUND 3
Model recovery begins
4 clean clients only. Accuracy jumps to 90.59% immediately after reset.
ROUNDS 4–15
Steady convergence
Attacker continues sending poison but is rejected every round. Trust score decays to 0.206. Final accuracy: 96.78%.

Key Observations

  • Attacker's cosine similarity stayed near 0.000 every round — poisoned weights have no correlation with honest updates
  • Trust score decayed from 1.0 → 0.206 over 15 rounds, demonstrating permanent reputation damage
  • After model reset in round 2, accuracy recovered to 90.59% in a single round
  • Final accuracy of 96.78% exceeded the unattacked baseline of 96.20%
  • All 61 verified updates are permanently recorded on the Ethereum blockchain with timestamps
  • Differential Privacy (ε=100) added negligible noise while protecting weight privacy
